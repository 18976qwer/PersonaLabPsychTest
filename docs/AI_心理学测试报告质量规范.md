# AI 心理学测试报告质量规范文档

## 一、文档概述

### 1.1 编制目的

本文档旨在为基于 MBTI 与九型人格的心理学测试报告 AI 生成系统建立统一的质量规范，确保 AI 生成的测试报告能够达到专业水准，满足用户体验需求，并确保内容的准确性、一致性与可读性。

### 1.2 适用范围

本规范适用于以下场景的 AI 内容生成：

- 测试报告各版块的描述性内容
- 针对不同人格类型的个性化分析
- 职业建议、人际关系指导等实用建议
- 报告总结与综合评估

### 1.3 核心原则

AI 内容生成应遵循以下核心原则：

| 原则 | 说明 |
|-----|------|
| **准确性** | 内容必须与测试结果（MBTI+九型人格）高度匹配，无类型混淆 |
| **特异性** | 描述应体现人格的独特性，避免泛化的模板化表述 |
| **完整性** | 内容数量和深度符合要求，信息量充足 |
| **一致性** | 格式、风格、语言在各版块间保持统一 |
| **安全性** | 内容不涉及心理诊断、医疗建议，不产生负面心理暗示 |
| **实用性** | 建议具体可操作，用户可应用于实际生活 |

## 二、问题现状与优化目标

### 2.1 当前存在的主要问题

经过对现有 AI 生成内容的分析，识别出以下 **七大核心问题**：

| 序号 | 问题类型 | 具体表现 | 严重程度 |
|:---:|---------|---------|:-------:|
| 1 | 内容过于简洁 | 缺乏深度和细节，信息量不足 | 高 |
| 2 | 人格匹配度低 | 描述与测试结果不符，出现类型混淆 | 极高 |
| 3 | 通用化严重 | 描述泛化，缺乏人格特异性 | 高 |
| 4 | 语言混杂 | 中英文夹杂，除专有名词外存在语言切换 | 中 |
| 5 | 内容重复 | 同一报告不同位置出现相同描述 | 高 |
| 6 | 专业壁垒高 | 术语晦涩，普通用户难以理解 | 中 |
| 7 | 数量缺失 | 要求返回 N 点内容，实际少于 N 点 | 高 |

### 2.2 潜在风险问题

除上述已识别问题外，还需关注以下 **潜在风险**：

| 序号 | 问题类型 | 风险说明 |
|:---:|---------|---------|
| 8 | 逻辑矛盾 | 不同版块间观点自相矛盾 |
| 9 | 理论偏差 | 引用的心理学概念不准确 |
| 10 | 格式混乱 | 标题层级、列表格式不统一 |
| 11 | 长度波动 | 不同人格类型的报告篇幅差异过大 |
| 12 | 过度推断 | 基于有限信息做出过于绝对的结论 |
| 13 | 情感缺失 | 内容过于理性，无法产生共鸣 |
| 14 | 模板化明显 | 用户感知到明显的模板填充痕迹 |
| 15 | 安全边界 | 对敏感心理话题的表述不当 |
| 16 | 行动指南缺失 | 建议过于理论化，缺乏可操作性 |
| 17 | 个体差异 | 忽略 MBTI×九型组合的独特性 |

### 2.3 优化目标总览

| 目标维度 | 具体要求 |
|---------|---------|
| **准确性** | AI 生成内容必须与测试结果 100% 匹配 |
| **特异性** | 每个版块的描述必须体现该人格类型的独特特征 |
| **完整性** | 内容详实、数量准确、论述充分 |
| **唯一性** | 同一报告内各版块内容无重复表述 |
| **一致性** | 格式、风格、深度在各人格类型间保持统一 |
| **语言纯净** | 根据页面语言提供纯正的单语言内容 |
| **通俗性** | 专业术语需附带通俗解释，确保可理解 |
| **实用性** | 建议和指导具体可操作 |
| **安全性** | 不涉及心理诊断，不产生负面暗示 |
| **伦理性** | 无类型偏见，无歧视性表述 |

## 三、报告结构规范

### 3.1 报告版块划分

测试报告共分为 **五个核心版块**：

| 序号 | 版块名称 | 核心内容 | 预估占比 |
|:---:|---------|---------|:-------:|
| 1 | 人格特质 | 核心性格、行为模式、认知方式 | 25% |
| 2 | 个人成长 | 成长建议、发展潜力、改进方向 | 20% |
| 3 | 职业路径 | 职业匹配、发展方向、行业推荐 | 20% |
| 4 | 人际关系 | 社交模式、关系建议、沟通策略 | 20% |
| 5 | 报告总结 | 综合分析、整体建议 | 15% |

### 3.2 版块内容独立性原则

各版块之间应保持 **内容独立性**，遵循以下原则：

| 原则 | 说明 |
|-----|------|
| **主题独立** | 每个版块聚焦于特定主题，不重复讨论同一论点 |
| **视角独特** | 从不同角度分析同一人格特征，避免视角雷同 |
| **论据独立** | 每个版块使用独立的例证和论据支持观点 |
| **表述差异** | 避免直接复制粘贴或改写其他版块的段落 |

### 3.3 版块间内容关联规则

| 关联类型 | 处理方式 |
|---------|---------|
| **概念引用** | 如需引用其他版块的内容，以简要提及为主，不重复展开 |
| **观点递进** | 后序版块可在先序版块基础上进行深化，但需添加新信息 |
| **结论呼应** | 报告总结可回顾各版块核心结论，但需重新组织表述 |

## 四、内容质量标准

### 4.1 准确性标准

#### 4.1.1 人格匹配要求

| 要求项 | 具体标准 |
|-------|---------|
| MBTI 类型匹配 | 描述必须与测试结果的 MBTI 类型（16种）完全一致 |
| 九型人格匹配 | 描述必须与测试结果的九型人格型号完全一致 |
| 组合效应 | 必须体现 MBTI×九型人格组合的独特特征 |
| 翼型/副型 | 如有相关信息，需纳入考量并体现 |

#### 4.1.2 禁止行为

| 禁止类型 | 具体表现 |
|---------|---------|
| 类型混淆 | 将 INFP 描述为 INFJ 的特征 |
| 泛化描述 | 使用"大多数内向的人都这样"等模糊表述 |
| 过度泛化 | 将适用所有类型的描述写入特定类型报告 |
| 矛盾描述 | 前文说某类型"善于社交"，后文说"偏好独处" |

### 4.2 特异性标准

#### 4.2.1 人格特征描述要求

每个版块的内容必须体现以下特质：

| 特质类型 | 说明 | 示例 |
|---------|------|------|
| **独特性** | 该人格区别于其他类型的特征 | INTJ 的"战略思维"区别于 INTJ 的"执行力" |
| **标签性** | 该人格的标志性行为模式 | ENFJ 的"利他倾向" |
| **差异性** | 与对立/相邻类型的核心区别 | INFP vs ISFP 的核心差异 |
| **深度性** | 超越表面特征的深层心理机制 | 深层动机、核心恐惧、核心渴望 |

#### 4.2.2 特异性检验方法

内容生成后，应通过以下问题进行检验：

- 这段描述是否适用于其他 15 种 MBTI 类型？
- 这段描述是否适用于其他 8 种九型人格？
- 用户是否能从描述中感受到"这就是我"？
- 这段描述是否可以被轻易替换为其他类型？

### 4.3 完整性标准

#### 4.3.1 内容数量要求

| 场景 | 要求 |
|-----|------|
| 要求返回 N 点内容 | 必须准确返回 N 点，偏差不超过 ±1 |
| 要求返回 M 个维度 | 必须完整覆盖所有 M 个维度 |
| 组合型要求 | 每个子项都必须完整覆盖 |

#### 4.3.2 内容深度要求

每个内容点应包含以下层次：

| 层次 | 说明 | 字数参考 |
|-----|------|:-------:|
| **概念定义** | 是什么：该特征的心理学定义 | 30-50 字 |
| **行为表现** | 怎么样：在日常生活中的具体表现 | 80-120 字 |
| **深层原因** | 为什么：背后的心理动机和机制 | 60-100 字 |
| **应用建议** | 怎么做：对用户的实际指导意义 | 50-80 字 |

#### 4.3.3 字数标准与 UI 适配

| 版块 | 建议字数范围 | UI 适配说明 |
|-----|:-----------:|------------|
| 人格特质 | 800-1200 字 | 需适配卡片布局，单卡片文本不宜过长 |
| 个人成长 | 600-900 字 | 列表项文本应精炼 |
| 职业路径 | 600-900 字 | 避免长段落堆砌 |
| 人际关系 | 600-900 字 | 分点清晰 |
| 报告总结 | 400-600 字 | 概要性强 |
| **单份报告总计** | **3000-4600 字** | **注意：Prompt 中的微观字数限制（如 "≤30字"）具有最高优先级，以防止 UI 布局错乱。** |

### 4.4 唯一性标准

#### 4.4.1 重复检测规则

| 检测维度 | 标准 |
|---------|------|
| 句子级重复 | 相同或高度相似（>80%）句子不超过 2 处 |
| 段落级重复 | 相同或高度相似段落不得出现 |
| 观点级重复 | 核心观点的重复表述不得超过 2 次 |
| 例证重复 | 生活实例、行为描述不得完全相同 |

#### 4.4.2 跨版块独立性要求

| 版块组合 | 允许重叠内容 | 禁止重叠内容 |
|---------|------------|-------------|
| 人格特质 ↔ 个人成长 | 特征概念可提及 | 成长建议不可重复 |
| 职业路径 ↔ 人际关系 | 性格特点可引用 | 具体建议不可雷同 |
| 各版块 ↔ 报告总结 | 核心结论可总结 | 具体论述不可复制 |

### 4.5 一致性标准

#### 4.5.1 格式一致性

| 格式元素 | 标准 |
|---------|------|
| 数据结构 | 严格遵循定义的 JSON Schema |
| 列表格式 | 使用 JSON Array 存储列表项 |
| 标点符号 | 中文版使用中文标点，英文版使用英文标点 |
| 数字格式 | 全篇统一使用阿拉伯数字 |

#### 4.5.2 风格一致性

| 风格维度 | 标准 |
|---------|------|
| 语气 | 温和、专业、鼓励性，避免说教或贬低 |
| 人称 | 全篇统一使用第二人称"你"或"您" |
| 时态 | 描述人格特征使用现在时，展望未来使用将来时 |
| 措辞 | 避免口语化表达，保持书面语风格 |

#### 4.5.3 深度一致性

| 要求 | 说明 |
|-----|------|
| 各人格类型 | 不同 MBTI/九型类型的报告深度保持一致 |
| 各版块 | 五个版块的论述深度保持相对均衡 |
| 相似特征 | 对相似人格特征的描述深度保持一致 |

## 五、语言规范

### 5.1 语言纯净性标准

#### 5.1.1 单语言原则

| 页面语言 | 输出语言要求 |
|---------|------------|
| 中文页面 | 全部内容使用中文，包括专有名词的中文翻译 |
| 英文页面 | 全部内容使用英文，包括专有名词的标准英文术语 |

#### 5.1.2 专有名词处理规则

| 类别 | 中文版处理 | 英文版处理 |
|-----|-----------|-----------|
| MBTI 类型 | 使用中文全称（如"内向-直觉-思考-判断"）或标准缩写（INTJ） | 使用标准英文缩写（INTJ） |
| 九型人格型号 | 使用中文数字（如"一号人格"）或标准说法（完美主义者） | 使用数字（Type 1）或标准英文名称（The Reformer） |
| 心理学术语 | 首次出现时给出中文译名和英文原名 | 使用标准英文术语 |
| 其他专有名词 | 遵循中文学术惯例 | 遵循英文学术惯例 |

#### 5.1.3 禁止行为

| 禁止类型 | 示例 |
|---------|------|
| 中英文混排 | "你的 INTJ 特质很明显"（应为"你的 INTJ 型特质"或"你的内向-直觉-思考-判断型特质"） |
| 夹带翻译 | "你是一个 perfectionist（完美主义者）" |
| 不必要的英文 | "你在九型人格中属于 Type 5 The Investigator"（中文版应写"五号人格——调查者"） |

### 5.2 通俗性标准

#### 5.2.1 术语解释规则

| 场景 | 处理方式 |
|-----|---------|
| 专业术语首次出现 | 必须附带通俗解释，格式如："内倾（Introversion，指能量来源于内部世界，偏好独处和反思）" |
| 心理学概念 | 需用生活化语言说明其含义和应用 |
| 人格类型名称 | 需解释该名称的含义和由来 |

#### 5.2.2 通俗化表述示例

| 原始表述 | 通俗化表述 |
|---------|-----------|
| "你的内倾特质使你倾向于从独处中获得能量" | "你是一个内向的人，相比于社交聚会，你更喜欢独自待着，这能让你感到精力充沛。" |
| "九型人格中的核心恐惧是害怕无意义" | "你最深的恐惧是觉得自己做的事情没有价值、不重要。" |
| "Fi 功能主导使你重视个人价值观的一致性" | "对你来说，做真实自己、遵守自己的价值观比什么都重要。" |

#### 5.2.3 例证要求

每个主要观点应配套至少一个生活化例证：

| 例证类型 | 说明 |
|---------|------|
| 日常生活 | "比如在聚会中，你可能会..." |
| 工作场景 | "在职场中，当你面对..." |
| 人际关系 | "和朋友相处时，你通常会..." |
| 决策场景 | "做重要决定时，你会..." |

### 5.3 语气规范

#### 5.3.1 正面导向

| 要求 | 说明 |
|-----|------|
| 积极肯定 | 强调人格优势，避免过度强调缺陷 |
| 成长导向 | 将"弱点"表述为"成长空间" |
| 鼓励性 | 使用支持性、赋能性的语言 |

#### 5.3.2 避免绝对化

| 禁止表述 | 建议表述 |
|---------|---------|
| "你一定是..." | "你倾向于..." |
| "你绝对是..." | "通常来说..." |
| "你永远不会..." | "你可能不太会..." |
| "这就是你的宿命" | "这是你的一种倾向，可以通过意识来调整" |

## 六、安全与伦理规范

### 6.1 内容安全准则

#### 6.1.1 禁止内容

| 类别 | 具体表现 |
|-----|---------|
| 心理诊断 | 不得将测试结果与心理疾病、障碍关联 |
| 医疗建议 | 不得建议用户寻求心理治疗或服药 |
| 负面标签 | 不得将任何人格类型描述为"问题"或"障碍" |
| 价值判断 | 不得暗示某类型"更好"或"更差" |
| 宿命论 | 不得暗示人格类型决定命运，不可改变 |

#### 6.1.2 敏感话题处理

| 话题 | 处理方式 |
|-----|---------|
| 心理健康问题 | 如用户有抑郁倾向等特征，措辞需格外温和，建议寻求专业帮助 |
| 社交焦虑 | 将其描述为"成长空间"，而非"问题" |
| 完美主义 | 同时强调其积极面（高标准）和需要注意的地方 |
| 人际冲突 | 提供建设性的改进建议，而非指责 |

#### 6.1.3 安全声明

每个报告应包含类似声明：

> "本报告基于人格类型理论，仅供参考，不代表心理诊断或医疗建议。每个人都是独特的，不应被人格类型所限制。如果您有任何心理困扰，建议咨询专业心理健康人士。"

### 6.2 伦理准则

#### 6.2.1 反偏见原则

| 原则 | 说明 |
|-----|------|
| 平等尊重 | 所有 16 种 MBTI 类型和 9 种九型人格都同等重要有价值 |
| 中立表述 | 避免使用带有情感色彩的词汇描述任何类型 |
| 无优劣之分 | 不得暗示外倾比内倾好、感觉比直觉好等 |

#### 6.2.2 个体差异原则

| 原则 | 说明 |
|-----|------|
| 组合独特性 | 强调 MBTI×九型人格组合的独特性 |
| 发展层级 | 同一类型在不同发展层级表现差异巨大 |
| 情境因素 | 人格表现受情境影响，非绝对固定 |

## 七、输出格式规范

### 7.1 JSON 结构化输出要求

本项目 AI 输出为 **严格的 JSON 对象**，而非 Markdown 文档。AI 必须严格遵循后端定义的 Schema 结构。

#### 7.1.1 数据结构完整性
- **严格匹配 Schema**：输出的 JSON 键值对（Key-Value）必须与 `promptTemplates.ts` 中定义的结构完全一致。
- **禁止 Markdown 标记**：JSON 字符串字段中**禁止**包含 Markdown 标题标记（如 `#`、`##`）或代码块标记（```json），除非特定长文本字段明确允许。
- **无多余字段**：不得返回未定义的字段。

#### 7.1.2 列表与数组
- **数组格式**：列表内容应作为 JSON Array 返回，而非在一个字符串中用“1. 2. 3.”拼接。
- **数量约束**：Array 的长度必须严格符合 Prompt 要求（如 `top3` 必须包含 3 个元素，`strengths` 必须包含 3 个对象）。

#### 7.1.3 文本字段格式
- **标点符号**：中文内容使用全角标点，英文内容使用半角标点。
- **换行处理**：仅在长文本字段（如“总结全文”）中使用 `\n` 进行换行，短文本/标题字段禁止换行。
- **首行缩进**：**不需要**在文本中手动添加缩进空格，由前端 CSS 统一控制。

### 7.2 格式校验标准

| 校验项 | 标准 |
|-------|------|
| JSON 合法性 | 必须是有效的 JSON 格式，无语法错误 |
| 字段完整性 | 所有必需字段（Required Fields）均存在且非空 |
| 类型匹配 | 字符串、数组、数字等类型必须与定义一致 |
| 长度合规 | 数组长度符合要求，字符串长度未超限 |

## 八、质量检验机制

### 8.1 自动化检验

#### 8.1.1 数量校验

| 校验内容 | 触发条件 | 处理方式 |
|---------|---------|---------|
| 内容点数 | 预期 N 点，实际 ≠ N | 返回要求重生成 |
| 维度覆盖 | 预期 M 个维度，缺失 | 返回要求补充 |
| 字数范围 | 超出设定范围 | 警告或要求调整 |

#### 8.1.2 重复校验

| 校验内容 | 阈值 | 处理方式 |
|---------|-----|---------|
| 句子相似度 | >80% | 标记并提示 |
| 段落相似度 | >70% | 标记并提示 |
| 观点重复 | 同一观点 >2 次 | 标记并提示 |

#### 8.1.3 语言校验

| 校验内容 | 触发条件 | 处理方式 |
|---------|---------|---------|
| 中英文混排 | 检测到夹带英文 | 返回要求纯化 |
| 专有名词错误 | 使用非标准译名 | 标记并修正 |
| 标点混用 | 中英文标点混用 | 返回要求统一 |
